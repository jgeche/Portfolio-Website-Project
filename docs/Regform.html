What is the project?

For the third project, I was asked to create my first ever online registration form that you see everywhere on the web. The form must have two views, both tablet and mobile views. The form must use correct form format in the HTML document and include the following (two form sections, fieldsets and legends, text input, email input, telephone input, select menu, checkboxes, radio buttons, text field and submit button). The form must also match the provided mockups of tablet and mobile views.

How was it put together?

First I created a simple form with content and form tags. I used a mobile first approach which I learned from the Responsive Layout Project. I used Chrome DevTools to preview the results of my CSS to correctly style the form in both views and match the mockups. All the correct form fields were added along with their corresponding IDs. I also added placeholder text in the name, address and textarea form-fields so the user would know which field is required by them to fill out.

What were the challenges?

I got my project back with a couple issues. The form structure needed some finessing as I put my form tag inside a fieldset and I only used one form tag. My form was heavy with div tags so I had to crop that up. I needed to also add normalize.css to my form and I had some spacing issues with the mockups as the label had to be on the left of the input for tablet view and one on top of the other for mobile. I went to work on these issues, adding another form field, fixing the fieldset and adding normalize.css. I then corrected the spacing issues so they matched the mockups and validated the code. The second time around the project was successful.
